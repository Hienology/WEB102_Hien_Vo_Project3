# WEB102_Hien_Vo_Project3

Project 3 is a React + Vite flashcard quiz game that extends Project 2 with answer input, correctness feedback, random card shuffling, and streak tracking.

## Features

- Flip each card to see question/answer content.
- Type an answer and submit to check if it is correct.
- Accept normalized answers (case-insensitive, punctuation-insensitive).
- Track both current streak and longest streak.
- Shuffle the deck order at runtime.
- Show difficulty badges (`easy`, `medium`, `hard`) with different accent colors.
- Handle image loading failures gracefully.

## Tech Stack

- React 19
- Vite 6
- JavaScript (ES Modules)
- CSS

## Getting Started

### 1. Install dependencies

```bash
npm install
```

### 2. Run locally

```bash
npm run dev
```

### 3. Build for production

```bash
npm run build
```

### 4. Preview production build

```bash
npm run preview
```

### 5. Build for GitHub Pages

```bash
npm run build:pages
```

This command builds the app and copies the output into the `docs/` folder format used for GitHub Pages deployment.

## Architecture Overview

### `src/main.jsx`
- App entry point.
- Mounts `App` inside React `StrictMode`.

### `src/App.jsx`
- Top-level state manager.
- Stores deck order, current card index, current streak, and longest streak.
- Handles navigation (`Prev`, `Next`), shuffling, and streak updates after answer submission.
- Passes data and callbacks to child components.

### `src/components/CardView.jsx`
- Renders the flashcard front/back UI.
- Handles flip behavior on click.
- Collects user guess and validates it against the expected answer.
- Uses normalized string comparison and accepts the primary answer form when parenthetical notes exist.
- Reports result back to `App` through `onSubmit`.

### `src/components/Controls.jsx`
- Renders navigation and shuffle controls.
- Disables `Prev` on first card and `Next` on last card.
- Shows index counter (`current / total`).

### `src/data/deck.js`
- Contains all quiz card content.
- Each card includes:
	- `frontText`
	- `backText`
	- `frontImage`
	- `backImage`
	- `category`

### `src/utils/normalize.js`
- Normalizes text for answer comparison.
- Current normalization strategy:
	- trim whitespace
	- convert to lowercase
	- remove punctuation

### `vite.config.js`
- Uses React plugin.
- Sets `base` path differently for local dev vs production build to support GitHub Pages project routing.

## Answer Validation Behavior

When a user submits an answer:

1. Input is normalized.
2. Correct answer is normalized.
3. A second accepted form is generated by removing parenthetical notes from `backText`.
4. If either normalized answer matches, the submission is treated as correct.
